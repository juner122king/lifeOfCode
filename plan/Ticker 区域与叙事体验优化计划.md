代码人生：Ticker 区域与叙事体验优化计划

 Context

 作为文字游戏，ticker 区域和文本叙事是核心体验。当前游戏虽有完整的数值系统，但缺乏足够的故事感、代入感和实时反馈。

 核心问题

 经过深度分析发现 10 个体验薄弱环节：

 🔴 高优先级（严重影响体验）
 1. Ticker 信息过于简洁 - 缺乏上下文，玩家不知道"这是否是预期状态"
 2. 实时反馈延迟感强 - 3秒更新周期太长，缺乏节奏感
 3. 建议系统过于被动 - 不解释"为什么"，无法优先级排序

 🟡 中优先级（影响理解）
 4. 警告等级区分不足 - 无法区分"预警"vs"已触发"
 5. 成就缺乏视觉高亮 - 里程碑事件容易被忽视
 6. 资源变化缺乏实时性 - 看不到"当前速率"和"效率对比"

 🟢 低优先级（体验优化）
 7. 无法追踪长期趋势 - 不知道"精力在恢复还是恶化"
 8. 命令反馈信息冗余 - 核心信息不突出
 9. 日程确认提示强硬 - 缺乏自由度
 10. 术语命名不统一 - 降低可读性

 叙事不足

 - 背景故事仅在创建时展示，游戏中很少提及
 - 属性增长无叙事解释 - 为什么沟通从4提升到20？
 - 关键节点文本简短 - 项目完成只列举奖励，缺少"客户反馈"
 - 活动执行过程透明不足 - 不知道为什么产出这么多/这么少
 - 没有"角色成长故事线" - 从菜鸟到专家缺乏叙事连贯性

 ---
 优化策略：三阶段实施

 Phase 1：增强 Ticker 实时性与上下文（核心体验）

 目标： 让玩家始终清楚"现在发生什么"和"接下来该做什么"

 Phase 2：丰富事件与叙事文本（代入感）

 目标： 让玩家感受到"我是这个角色"，而不是"我在操作数据"

 Phase 3：建立成长故事线（长期沉浸）

 目标： 让玩家体验"从新手到专家"的完整旅程

 优化 1.3：建议系统改为优先级推送

 文件： src/game.js (formatNextAdvice 函数)

 当前： 只显示单一建议
 建议：目标已完成，先 claim xxx 领取奖励。

 优化为： 多级建议 + 原因说明
 🔴 紧急：精力 8/100，继续活动将停止推进 → 建议：rest 或切换作息
 🟡 重要：2个目标可领取（+200金钱，+3声望） → 建议：claim all
 🔵 提示：Bug风险 78/100 偏高 → 建议：安排 bug-hunting 活动
 💡 推荐：当前空闲，可安排上午活动 → 建议：plan morning

 实现逻辑：
 function getAdviceList(state) {
   const advices = [];

   // 紧急级（阻断进度）
   if (state.resources.energy <= 10) {
     advices.push({
       level: "danger",
       emoji: "🔴",
       text: "精力耗尽，活动已停止",
       reason: "精力 <= 10",
       action: "rest 或 lifestyle health"
     });
   }

   // 重要级（高价值机会）
   const claimable = getClaimableGoals(state);
   if (claimable.length > 0) {
     const rewards = claimable.reduce((sum, g) => sum + (g.rewards.money || 0), 0);
     advices.push({
       level: "warn",
       emoji: "🟡",
       text: `${claimable.length}个目标可领取`,
       reason: `+${rewards}金钱等奖励`,
       action: "claim all"
     });
   }

   // 提示级（风险预警）
   if (state.resources.bugs > 70) {
     advices.push({
       level: "info",
       emoji: "🔵",
       text: "Bug风险偏高",
       reason: `当前 ${state.resources.bugs}/100`,
       action: "plan bug-hunting"
     });
   }

   // 推荐级（空闲建议）
   if (!state.activeActivityId && !state.waitingForSchedule) {
     advices.push({
       level: "hint",
       emoji: "💡",
       text: "当前空闲",
       reason: "可安排活动提升产出",
       action: "plan morning <activity>"
     });
   }

   return advices.slice(0, 3); // 最多显示3条
 }

 Phase 2: 丰富事件与叙事文本

 优化 2.1：为随机事件增加故事变体

 文件： src/content.js (randomEvents 数组)

 当前： 每个事件只有1条固定文本

   id: "requirement-change",
   name: "需求变更",
   message: "产品说只是改一个小需求，技术债增加了。"
 }

 优化为： 每个事件3-5条变体，随机抽取
 {
   id: "requirement-change",
   name: "需求变更",
   messages: [
     "产品经理周五 17:59 发来一句话：'只是改一个小需求，明天上线'",
     "客户临时提了个新想法，听起来很简单，实际上要重构半个模块",
     "刚做完的功能被推翻了，原因是'老板觉得应该换个颜色'",
     "需求文档又双叒叕改了，而你昨天刚写完这部分代码",
     "产品说这个需求'很简单'，但你看到实现方案时沉默了"

   apply(state) {
     state.resources.techDebt += 10;

 优化 2.2：活动执行中增加阶段性叙事

 文件： src/game.js (settleActivity 函数)

 当前： 活动执行过程无文本反馈，只在结束时显示数值

 优化为： 每隔一定时间推送阶段文本

 示例（写功能活动）：
 [00:15] 你打开编辑器，新建了一个组件文件
 [00:45] 代码逐渐成型，但你意识到这个设计可能有性能问题
 [01:30] 终于跑通了测试用例，不过留下了几个TODO注释
 [02:00] 功能基本完成，提交了PR等待Review

 实现方式：
 - 为每个活动定义 narrativeStages 数组
 - 根据活动时长触发对应阶段的文本
 - 文本内容反映活动产出和风险

 activity({
   id: "feature-coding",
   narrativeStages: [
     { seconds: 900, texts: [
       "你打开编辑器，开始构思组件结构",
       "敲下第一行代码，感觉今天手感不错",
       "从GitHub上找到了一个可以参考的实现"
     ]},
     { seconds: 2700, texts: [
       "代码逐渐成型，但你意识到这个设计可能有坑",
       "遇到了一个意料之外的边界情况",
       "不断调试，终于找到了问题所在"
     ]},
     { seconds: 5400, texts: [
       "功能基本完成，开始写单元测试",
       "提交了PR，等待同事Review",
       "回头看代码，留下了几个TODO注释"
     ]}
   ]
 })

 Phase 3: 建立角色成长故事线

 优化 3.1：属性提升触发成长事件

 文件： src/game.js (addAttributeExp 函数)

 当前： 属性增长只更新数值，无文本反馈

 优化为： 每提升 20 点触发成长事件

 示例（逻辑属性 20→40）：
 [成长] 逻辑思维提升

 在反复的代码Review中，你逐渐学会了：
 - 从系统的角度思考问题
 - 预见潜在的边界情况
 - 权衡不同方案的利弊

 队友评价："你提的问题越来越尖锐了"

 逻辑 20 → 40

 实现方式：
 - 在 addAttributeExp 中检测属性跨越阈值（20, 40, 60, 80）
 - 推送对应的成长事件
 - 为每个属性x阶段定义专属文本

 技术实现要点

 1. 叙事文本数据结构

 // 活动叙事阶段
 narrativeStages: [
   { seconds: 900, texts: ["文本1", "文本2", "文本3"] }
 ]

 // 随机事件变体
 messages: ["变体1", "变体2", "变体3"]

 // 人物成长故事
 growthStoryline: {
   "节点名": { trigger: {条件}, text: "文本" }
 }

 2. 叙事触发机制

 - 时间触发： 活动执行到特定秒数
 - 阈值触发： 属性/资源跨越阈值
 - 事件触发： 项目完成、技能达成、职业晋升

 3. 文本选择策略

 - 随机选择： 从变体数组中随机抽取
 - 权重选择： 根据当前状态选择最契合的文本
 - 人物特化： 根据人物卡ID选择专属文本
