# 代码人生

《代码人生》是一个纯文字放置类 Node CLI/TUI 游戏原型。你从一张人物卡和一份程序员档案开始，在活动、学习、工具、项目、目标和职业晋升之间分配时间，把资源和能力一步步滚起来。

## 功能概览

- 档案与人物卡：首次创建档案时选择不同背景的人物卡，获得不同初始属性、资源、技能和活动等级。
- 三阶段日程：每天 09:00 暂停，确认上午、下午、晚上计划后按真实时间自动执行。
- 持续作息：休整窗口自动按当前作息基调结算，切换只会在次日 09:00 生效。
- 活动推进：把写功能、系统学习、排查 Bug、重构、写测试、写文档等活动安排进日程来产出资源。
- 技能成长：学习并升级技能，解锁新项目和更高阶的活动收益。
- 工具购买：用金钱购买工具，改善产出、风险或压力表现。
- 项目交付：从委托板接项目，按阶段推进并随进度消耗素材，获得金钱、声望、技能经验和属性成长。
- 目标与晋升：领取主线和支线目标奖励，满足条件后从实习程序员逐步晋升。
- CLI/TUI 双入口：支持传统命令行交互，也支持 Ink 构建的终端 UI。

## 快速开始

```bash
npm install
npm run tui
```

也可以使用传统 CLI：

```bash
npm start
```

运行测试：

```bash
npm test
```

## 基础玩法

首次启动时需要选择人物卡来创建档案。人物卡会影响初始属性、资源、技能、活动等级和开局路线。

游戏按固定日程推进：上午 09:00-12:00、下午 14:00-18:00、晚上 18:00-21:00。12:00-14:00、晚上 `none` 的 18:00-21:00、21:00 到次日 09:00 会自动按当前作息基调休整。每天 09:00 会让明日作息生效、刷新委托板，并暂停等待玩家安排今日日程；上午、下午必填，晚上可以安排任务或 `none` 晚间放松。日程确认后会预扣学习资源；项目素材不在确认或接取时预扣，而是在阶段推进中按进度消耗，并且当天日程不可修改。

作息基调通过 `lifestyle [id]` 查看或设置。默认 `health`，也可以选择 `tech_surfing`、`cyber_gaming` 或 `side_hustle`。精力固定为 0-100 的硬资源；活动、技能学习和项目都会消耗精力，精力耗尽时停止推进。精力只会在休整中逐步恢复，基础恢复深夜最高，实际恢复会被当前压力压低；`side_hustle` 是高风险副业路线，深夜会即时消耗精力换取少量金钱和声望，并增加压力。

主要成长循环是：安排活动获得资源，积累知识和金钱学习技能，购买工具改善效率或风险，接取里程碑或随机委托并分阶段交付，领取目标奖励，再满足条件尝试晋升。

## 常用命令

| 命令 | 说明 |
| --- | --- |
| `status` | 查看当前档案、资源、日程、活动、学习、项目和职业状态 |
| `plan` | 查看今日日程草稿或已确认日程 |
| `plan morning activity <id>` | 把活动安排到上午 |
| `plan afternoon skill <id>` | 把技能学习安排到下午 |
| `plan evening project <id>` | 把项目安排到晚上，晚上任务按加班惩罚结算 |
| `plan evening none` | 晚间放松，晚上不安排任务，避免加班压力，并按当前作息基调进行轻量恢复 |
| `plan confirm` | 确认今日日程，确认后不可修改 |
| `plan clear` | 清空未确认的今日日程草稿 |
| `activities` | 查看活动列表、等级、解锁状态和当前状态 |
| `start <activityId>` | 旧入口，只提示改用 `plan` |
| `stop` | 暂停当前底层活动、学习或项目，主要用于调试 |
| `learn <skillId>` | 旧入口，只提示改用 `plan` |
| `upgrade <skillId>` | 升级已学技能 |
| `buy <toolId>` | 购买工具 |
| `projects` | 查看当前委托板，包含未完成/进行中的里程碑和每日随机委托 |
| `project <projectId>` | 接取、继续或重复提交项目；直接使用已知项目 ID 仍然可用 |
| `goals` | 查看目标进度 |
| `claim [id\|all]` | 领取指定目标或领取所有当前可领取目标 |
| `promote` | 尝试职业晋升 |
| `week <id>` | 设置周重点，可选 `learning`、`project`、`freelance`、`quality`、`balanced` |
| `lifestyle [id]` | 查看或设置明日作息，可选 `health`、`tech_surfing`、`cyber_gaming`、`side_hustle` |
| `profiles` | 查看档案列表 |
| `profile ...` | 新建、加载、保存、重命名或删除档案 |
| `cards` | 查看可选人物卡 |
| `events` | 查看当前世界日历和事件 |
| `wait <seconds>` | 在 CLI 中快进指定秒数 |
| `list <type>` | 查看内容列表 |
| `help` | 查看帮助 |
| `save` | 保存当前档案 |
| `quit` / `exit` | 保存并退出 |

## 项目结构

```text
src/
  content.js      游戏内容配置入口：职业、人物卡、活动、技能、工具、目标、随机事件
  content/        内容构建器和项目/委托配置
  game.js         核心状态、结算、命令处理、存档和 CLI 入口
  tui.js          Ink TUI 入口和交互界面
  tuiTheme.js     TUI 配色、状态色和进度条渲染工具
test/             Node 内置 test runner 测试
docs/             玩法设计和人物卡说明
```

存档默认写入仓库根目录下的 `.save/`，该目录不会提交到版本库。

## 开发说明

项目使用 Node.js 和 CommonJS 模块。TUI 基于 `ink`、`ink-spinner` 和 `react`。

核心测试命令：

```bash
npm test
```

测试文件位于 `test/`，使用 Node 内置 test runner（`node --test`）。
