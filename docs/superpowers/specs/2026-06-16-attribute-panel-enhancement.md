# 属性面板展示优化设计

## 一、设计目标

优化属性面板展示，帮助玩家在规划成长路线时：
1. 快速浏览所有属性的进度和下一目标
2. 深入查看单个属性的详细信息和收益
3. 了解里程碑解锁情况和未来目标

## 二、使用场景

**主要场景**：玩家规划属性成长路线
- 决定下一步提升哪个属性
- 了解某个属性到下一里程碑还需多少
- 查看某个属性当前提供的具体收益

## 三、设计方案：分层折叠式

### 3.1 命令结构

| 命令 | 功能 | 示例 |
|------|------|------|
| `attr` 或 `attributes` | 显示所有属性摘要（默认视图） | `attr` |
| `attr <属性名>` | 显示单个属性详情 | `attr logic`、`attr 逻辑` |
| `attr milestones` | 显示所有属性里程碑总览 | `attr milestones` |

### 3.2 摘要视图（默认）

**目标**：一屏显示所有属性的关键信息

**显示内容**：
- 6个属性的一行摘要
- 当前等级 + 进度条（到下一里程碑）
- 经验进度（到下一级）
- 下一个里程碑名称
- 最近解锁提示

**示例输出**：
```
========== 属性总览 ==========

逻辑 42 [████████░░] 55    经验 240/396 (61%)    下一里程碑: Lv.55 质量守护者
专注 38 [███████░░░] 40    经验 85/147 (58%)     下一里程碑: Lv.40 持续输出
学习 25 [██████░░░░] 40    经验 12/147 (8%)      刚解锁: Lv.25 快速学习 ✓
沟通 18 [███░░░░░░░] 25    经验 45/84 (54%)      下一里程碑: Lv.25 有效表达
抗压 22 [████░░░░░░] 25    经验 20/96 (21%)      下一里程碑: Lv.25 压力管理
创造 20 [███░░░░░░░] 25    经验 55/90 (61%)      下一里程碑: Lv.25 创意火花

> 输入 attr <属性名> 查看详情 (如: attr logic 或 attr 逻辑)
> 输入 attr milestones 查看所有里程碑总览
```

**设计要点**：
- 进度条：显示从当前等级到下一个里程碑的距离
- 经验：显示到下一级的精确进度
- 最近解锁：如果本次会话解锁了里程碑，用 ✓ 标记
- 进度条长度：10个字符，视觉清晰

### 3.3 详情视图（单个属性）

**目标**：显示单个属性的完整信息，帮助玩家理解投资回报

**显示内容**：
1. 基本信息（当前等级、经验、有效值）
2. 已解锁里程碑（含完整叙事）
3. 下一里程碑（突出显示，含预估）
4. 未来里程碑（快速预览）
5. 当前收益（分基础、里程碑、总计）

**示例输出**：
```
========== 逻辑 详情 ==========

当前等级：42/100
经验进度：240/396 (61%) 到 Lv.43
有效属性：42.0 (基础 42 + 突破 0)

【已解锁里程碑】
  ✓ Lv.25 代码直觉觉醒
     Bug 风险额外 -5%
     你开始能预见代码中潜伏的问题，Bug 不再只是运行时的惊喜。
  
  ✓ Lv.40 架构洞察
     技术债降低效率的影响额外 -10%
     你开始能在复杂系统中看到隐藏的债务链条，重构不再只是凭感觉。

【下一里程碑】
  Lv.55 质量守护者 - 项目成功率额外 +8%
  还需：13 属性点 (约 1170 经验)
  预计：质量不再是事后补救，而是贯穿始终的设计决策。

【未来里程碑】
  Lv.70 系统思维 - 所有质量活动效率额外 +15%
  Lv.85 逻辑大师 - 加班时 Bug/债务风险额外 -20%

【当前收益】
基础属性加成（逻辑 42 带来）：
  • 质量活动效率：+17%
  • Bug 风险缓解：-13%
  • 技术债效率影响：-12%
  • 加班 Bug/债务风险缓解：+13%
  • 项目成功率：+5%

里程碑额外加成：
  • Bug 风险：额外 -5% (代码直觉觉醒)
  • 技术债效率影响：额外 -10% (架构洞察)

总计效果：
  • Bug 风险缓解：-18% (基础 -13% + 里程碑 -5%)
  • 技术债效率影响：-22% (基础 -12% + 里程碑 -10%)
  • 质量活动效率：+17%

> 输入 attr 返回属性总览
```

**设计要点**：
- 里程碑分三部分：已解锁（完整信息）、下一个（重点）、未来（简略）
- 下一里程碑显示精确的距离（属性点 + 经验预估）
- 收益分三层：基础加成、里程碑加成、总计（清晰展示叠加）
- 用 • 标记每个收益项，便于扫描

## 五、技术实现

### 5.1 文件修改清单

| 文件 | 改动内容 |
|------|---------|
| `src/game.js` | 新增 `getAttributeDetails()`、`getAttributeSummary()`、`getMilestoneOverview()` 函数 |
| `src/game.js` | 新增 `calculateAttributeBenefits()` 辅助函数（计算收益列表） |
| `src/game.js` | 导出新函数到 module.exports |
| `src/tui.js` | 新增 `renderAttributeSummary()`、`renderAttributeDetails()`、`renderMilestoneOverview()` |
| `src/tui.js` | 在 `processCommand()` 中添加 `attr` 命令处理 |

### 5.2 核心函数实现要点

**getAttributeDetails(state, attrId)**：
- 从 `state.attributes[attrId]` 获取当前等级
- 从 `state.attributeExp[attrId]` 获取当前经验
- 调用 `getAttributeUpgradeCost()` 计算下一级所需经验
- 从 `state.unlockedMilestones[attrId]` 获取已解锁里程碑
- 从 `ATTRIBUTE_MILESTONES[attrId]` 获取所有里程碑定义
- 调用 `calculateAttributeBenefits()` 计算当前收益

**calculateAttributeBenefits(state, attrId)**：
- 计算基础属性加成（调用现有的 `attributeBonus()` 函数）
- 获取里程碑加成（调用 `getMilestoneBonus()`）
- 组合成收益列表，区分基础、里程碑、总计

**renderAttributeSummary(summary)**：
- 渲染6行属性摘要
- 每行：名称 + 等级 + 进度条 + 经验 + 下一里程碑
- 底部提示如何查看详情

**renderAttributeDetails(details)**：
- 分5个部分渲染：基本信息、已解锁、下一个、未来、收益
- 使用颜色区分：绿色（已解锁）、黄色（下一个）、灰色（未来）
- 收益部分用缩进和 • 标记

## 六、用户体验改进

### 6.1 快捷操作

- 支持中英文属性名：`attr logic` 和 `attr 逻辑` 都能识别
- 自动补全：`attr l` 能识别为 `logic`（如果唯一匹配）
- 错误提示：输入错误属性名时，显示可用列表

### 6.2 视觉优化

- 进度条：用不同字符表示进度（█ 已完成，░ 未完成）
- 颜色：绿色（已解锁）、黄色（接近）、灰色（未来）
- 对齐：数字右对齐，标签左对齐，便于扫描

### 6.3 上下文提示

- 摘要视图底部：提示如何查看详情
- 详情视图底部：提示如何返回摘要
- 里程碑总览底部：提示如何查看详细收益
